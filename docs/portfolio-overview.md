# Shopifyエンジニア転職用ポートフォリオ設計書

## 1. ポートフォリオ全体のコンセプト

### コンセプト
**「Shopifyテーマ実装・ストア設計・運用理解まで見せる、実務寄りのECポートフォリオ」**

このポートフォリオは、単なるWeb制作作品集ではなく、Shopify制作会社・EC支援会社の採用担当者に対して、以下を伝えるためのサイトです。

- Shopifyテーマを触れるだけではなく、ストア構成を考えられる
- 商品・コレクション・メタフィールド・FAQ・Journalなど、ECサイト運用に必要な要素を理解している
- Riseテーマをベースに、ブランドに合わせたUI/UX改善ができる
- GitHub / Shopify CLI / 開発ストア / 実装レポートを使い、実務に近い流れで制作できる
- 未経験でも、制作会社の現場に入りやすい基礎体力がある

### メインメッセージ
**Shopifyテーマ実装からEC運用設計まで、実務フローを意識して学習・制作しています。**

### サイトの立ち位置
- Shopifyエンジニア転職用ポートフォリオ
- 静的HTML/CSS/SCSS/JavaScriptで構築
- GitHub Pagesで公開
- Shopifyデモストアへの導線を明確化
- 各Worksに「実装内容」「設計意図」「使用技術」「課題と改善」を掲載

### ターゲット
- Shopify制作会社
- EC支援会社
- D2Cブランド支援会社
- Web制作会社のShopify担当
- 未経験エンジニア採用担当者

## 2. 採用担当者に伝えるべき強み

### 1. Shopify特化で学習・制作している
一般的なHTML/CSSのポートフォリオではなく、Shopifyテーマカスタマイズ・商品設計・メタフィールド・コレクション設計まで学習している点を強調する。

### 2. テーマ実装だけでなく、EC運用視点がある
商品CSV、FAQ、Journal、商品導線、コレクション分類など、運用担当者やクライアントが使う部分を意識している点を出す。

### 3. 既存テーマをベースに改善できる
Riseテーマをベースに、ブランドごとの世界観や情報設計に合わせてカスタマイズできることを見せる。

### 4. ドキュメント化できる
README、CASE_STUDY、実装レポート、設計メモを用意し、制作プロセスを言語化できることを示す。

### 5. 現職で培った正確性・継続力を開発に転用できる
請求事務・物流業務で培った正確な処理、差異確認、期限遵守、業務改善視点をShopify実装にも活かせる。

## 3. 見せ方の基本方針

### デザイン方針
- 落ち着いた実務寄り
- 白・生成り・チャコール・薄いグレーを基調
- アクセントにShopifyらしいグリーンを控えめに使用
- 派手なアニメーションより、読みやすさと信頼感を優先
- PC/SPで採用担当者が確認しやすい構成

### トーン
- 「すごい作品を作りました」ではなく「実務を想定して設計・実装しました」
- 未経験であることは隠さず、学習プロセスと再現性を見せる
- 技術名の羅列ではなく、なぜ実装したかを説明する
